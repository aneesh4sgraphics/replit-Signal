What to do now (before data arrives)

You don’t need a Coach Console yet, but you do need the right exhaust.

Tell Replit to make sure these are quietly logged from Day 1:

1. Event-level logging (non-negotiable)

Every NOW MODE interaction should emit an event like:

now_card_served

now_action_completed

now_action_skipped

now_session_paused

now_session_idle_popup_shown

now_session_idle_dismissed

Each event should include:

user_id

customer_id

task_type (call / follow-up / enablement / hygiene)

outcome

time_to_action (seconds)

day_index (1–10)

efficiency_delta

You don’t show this yet. You just collect it.

2. Define success metrics early (even if hidden)

Internally agree on 5 numbers that matter. Everything else is noise.

I’d recommend:

Activation: % of users who complete ≥3 tasks/day

Completion rate: Avg tasks completed (out of 10)

Skip rate

Median time to first action

Conversion assist rate (orders or replies within X days after NOW actions)

Tell Replit:

We don’t need dashboards yet, but store everything needed to compute these later.

3. Lock the NOW MODE contract (no feature creep)

Write this as a hard rule for the system:

NOW MODE is the only place reps should work from day to day.
Everything else is reference, not workflow.

This protects you from:

“Can we add bulk actions?”

“Can we let reps pick tasks?”

“Can we show all customers?”

Those kill adoption.

What to watch for in the first 7–14 days

Once data flows, look for patterns, not opinions.

Green flags

Reps complete 7–10 tasks/day without reminders

Time to first action < 3 minutes

Low skip rate on calls

Efficiency score slowly trending up

Red flags (these guide Coach Console design later)

High skip rate on one task type → that task needs reframing

Long idle time before first action → onboarding issue

High activity, low outcomes → coaching/scripts problem

Low activity across the board → task mix or daily cap wrong

You don’t need a Coach Console to see these at first — even raw logs + SQL will tell the story.
