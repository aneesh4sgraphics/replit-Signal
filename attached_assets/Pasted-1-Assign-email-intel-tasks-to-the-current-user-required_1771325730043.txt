1) Assign email-intel tasks to the current user (required)

Fix Email Intelligence → FollowUpTask creation:

When creating follow-up tasks from email events, always set:

assignedTo = emailMessage.userId (the Gmail account owner that was synced)

OR fallback: assignedTo = customer.salesRepId if you prefer customer ownership

Rule: Email-intel tasks must never be unassigned.

Implementation

In email-event-extractor.ts (or wherever followUpTasks are created), add:

assignedTo

createdBy

sourceType = "email_event"

Also ensure:

customerId is present when matched

contactEmail stored in payload even if customer is not matched

2) Add a “manual linking” Spotlight card for unmatched emails (required)

If customer matching confidence = 0 (unmatched), don’t drop it.

Create a Data Hygiene Spotlight card:

Card type: LINK_EMAIL_TO_CUSTOMER
Action: show the email + suggested customer matches (by domain/company name)
Outcomes:

Linked ✅ → then generate the normal email follow-up task (assigned)

Not a customer 🚫 → mark ignored

Remind later ⏸ → snooze

This ensures email intelligence still surfaces even when matching is imperfect.

3) Expand Spotlight selection rules so email tasks show up (important)

Right now Spotlight likely only includes:

overdue follow-ups

calls

hygiene

outreach

Email intelligence tasks should count as Follow-ups or Outreach.

Selection logic change

Include email-intel tasks even if not overdue:

include tasks due today OR within next 1–3 days

prioritize by event severity (PO > urgent > approval > samples > objections > stale thread)

Otherwise you’ll only see email intel days later, which makes it feel “broken”.

4) Backfill existing email tasks that are currently invisible (important)

After the fix, you’ll still have old tasks sitting unassigned.

Add a one-time backfill script:

find followUpTasks where sourceType="email_event" and assignedTo is null

join back to the originating email/user

update assignedTo

This will immediately make existing intel appear in Spotlight.

5) Make Spotlight show “Email Signal” content inside the same card (UX)

Do NOT make a separate Email Intelligence section.

In Spotlight cards sourced from email:
Show:

Subject + date

1–2 key extracted phrases (why it triggered)

Event label (PO / approval / urgent / objection)

Suggested reply / script snippet (short)

Outcome buttons (example):

“Replied” (logs reply + schedules next check)

“Called” (logs call + schedules follow-up)

“Snooze”

“Not relevant”

This makes email intelligence feel real and actionable.
